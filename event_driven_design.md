# Описание Event-Driven архитектуры
Применительно к моему варианту задания главными событиями являются создание пользователя, создание аренды и завершение аренды. 

Производителями событий являются сервисы `clientService` и `rentService`.

Потребителями событий являются сервисы уведомления пользователей `userNotificationService` и бизнес аналитики `analyticsService`.

- EventAddUser: при создании пользователя мы уведомляем `userNotificationService` для отправки сообщения об успешной регистрации на почту и `analyticsService` для обновления статистики зарегистрированных пользователей по стране.
- EventAddRent: при создании аренды мы уведомляем `userNotificationService` для отправки сообщения об успешном создании аренды на почту и `analyticsService` для обновления статистики по арендованному автомобилю.
- EventFinalizeRent: при завершении аренды мы уведомляем `userNotificationService` для отправки сообщения о завершении аренды на почту.
